Angular scroll behavior problem

By default, Angular keeps the scroll position between route changes. This can be changed with scrollPositionRestoration: 'enabled' in the routing module.

However, if you set scroll-behavior: smooth on the html element in CSS, this behavior can be weird and sometimes it doesn't scroll to the top when changing routes too quickly. For this reason, it would be a good idea not to enable smooth scrolling in CSS. When the user navigates to a different page, more often than not, we don't want to have smooth scrolling to the top.

But what if we want smooth scrolling in the case of anchor links? We can solve it with a custom directive!
